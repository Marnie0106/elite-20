## 🔄 完整 KSTAR 循环

### K - Knowledge（知识收集）

在开始写文章之前，先通过多个 AI 工具收集专业知识：

1. **电子信息工程的核心就业方向**：嵌入式开发、信号处理、通信系统、物联网、FPGA
2. **AI 在编程学习中的应用**：语法学习、代码调试、项目答疑、练习规划
3. **AI 在嵌入式开发中的应用**：代码生成、文档阅读、开发板调试
4. **AI 在就业准备中的应用**：简历优化、面试对标、技能对标

**使用工具**：ChatGPT、DeepSeek、Claude（VS Code 内置）

---

### S - Situation（情境分析）

| 项目 | 具体情况 |
|------|----------|
| 身份 | 电子信息工程专业学生 |
| AI 基础 | 会用 ChatGPT 和 DeepSeek 进行基本对话，刚接触 Claude |
| 编程水平 | 正在重新学习 C 语言和 Python |
| 可用时间 | 约 2 小时 |
| 目标 | 产出一篇有实际价值的专业相关文章 |
| 平台 | GitHub Codespaces（在线开发环境） |
| 工具 | Claude（VS Code 聊天面板）、Git（代码版本管理） |

---

### T - Task（任务定义）

撰写一篇 1500-2000 字的文章：
- 主题明确：AI 工具 + 电子信息工程
- 结构清晰：引言 → 正文（3模块） → 总结
- 内容实用：具体工具 + 使用场景 + 操作提示
- 体现 KSTAR 方法论的应用过程
- 同时在 Codespaces 中完成项目初始化和 Git 版本管理

---

### A - Action（执行过程）

#### 第一轮：知识收集与工具调研

**Step 1 — 用 ChatGPT 收集专业知识**

提示词：
```
电子信息工程专业的核心就业方向有哪些？每个方向需要什么技能？
```

得到嵌入式开发、信号处理、通信系统、物联网、FPGA 等方向信息。

**Step 2 — 用 DeepSeek 调研 AI 工具应用**

提示词：
```
AI 工具如何帮助电子信息工程学生？请从编程学习、项目开发、求职三个维度分别说明。
```

得到 AI 辅助编程学习、AI 加速项目开发、AI 提升就业竞争力三个维度的详细回答。

#### 第二轮：使用 Claude（VS Code）进行项目操作

**Step 3 — 打开 GitHub Codespaces**

登录 GitHub，进入 elite-20 项目，点击 Codespaces 打开在线开发环境。在 VS Code 右侧的聊天面板中与 Claude 对话。

提示词：
```
帮我理解这个 elite-20 项目的整体结构，每个文件夹是做什么的？
```

Claude 回复了项目结构分析：
- `agents/` — AI 智能体配置
- `skills/` — 技能模块
- `commands/` — 命令定义
- `docs/` — 文档
- `src/` — 源代码
- `CLAUDE.md` — Claude 配置文件

**Step 4 — 用 Claude 修改项目文件**

提示词：
```
帮我修改 README.md，加上一段关于我对这个项目的理解和对 AI+X 课程的期待。
```

Claude 生成了修改建议，我确认后应用到文件中。

#### 第三轮：文章生成与迭代

**Step 5 — 用 ChatGPT 生成初稿**

提示词：
```
请帮我写一篇文章，主题是《电子信息工程学生必备的 AI 工具链——从编程到智能开发》。

作者背景：电子信息工程专业学生，正在重新学习 C 语言和 Python，希望用 AI 提升学习效率和就业竞争力。

要求：
1. 结构清晰，有引言、正文（3-4个板块）、总结
2. 每个板块要有具体的 AI 工具推荐和使用场景
3. 覆盖以下方向：编程学习（C/Python）、嵌入式开发、信号处理、就业准备
4. 语言通俗易懂，面向同专业同学
5. 字数 1500-2000 字
6. 要有实操建议，不要空谈理论
```

**第一次结果**：文章结构完整，但内容偏向泛泛，缺少具体的操作步骤和工具对比。

**Step 6 — 迭代优化（DeepSeek）**

提示词：
```
这篇需要改进：
1. 每个板块增加具体的提示词示例，让读者知道怎么实际去用 AI
2. 增加一个 AI 工具对比表格（ChatGPT vs DeepSeek vs Claude 各自适合什么场景）
3. 总结部分要更加升华，强调方法论的重要性
```

**第二次结果**：明显改善，增加了提示词示例和工具对比表。

**Step 7 — 最终校对（人工 + Claude）**

把文章导入 Codespaces，用 Claude 检查语句不通顺的部分：

提示词：
```
请帮我检查这篇文章，找出语句不通顺、逻辑不连贯的地方，给出修改建议。
```

根据 Claude 的建议做了最后修改。

#### 第四轮：Git 版本管理

**Step 8 — 提交代码到 GitHub**

在 Codespaces 终端中执行：

```bash
# 第1次提交：初始化 KSTAR 记录
git add .
git commit -m "初始化KSTAR循环记录"

# 第2次提交：补充文章和Claude使用记录
git add .
git commit -m "补充文章内容，增加Claude使用过程"

# 第3次提交：最终完善
git add .
git commit -m "优化文章内容，完成KSTAR循环"

# 推送到远程仓库
git push
```

### R - Result（结果反思）

**✅ 产出成果：**
- 完整文章：《电子信息工程学生必备的 AI 工具链》（见 `01-understanding/`）
- KSTAR 完整循环记录（本文件）
- elite-20 项目在 Codespaces 中完成初始化和文件修改
- Git 提交记录3次，记录了完整的修改过程

**✅ 使用的工具汇总：**

| 工具 | 用途 | 效果 |
|------|------|------|
| ChatGPT | 知识收集、初稿生成 | 速度快，适合发散思维 |
| DeepSeek | 工具调研、迭代优化 | 中文理解好，建议具体 |
| Claude（VS Code） | 项目理解、文件修改、文章校对 | 集成在编辑器中，操作方便 |
| GitHub Codespaces | 在线开发环境 | 免配置，打开即用 |
| Git | 版本管理和代码提交 | 记录了完整的修改历史 |

**⚠️ 遇到的问题：**
- 第一次 ChatGPT 生成的文章过于泛泛，通过迭代提示词改进了针对性（改了2次才满意）
- Codespaces 初次使用不太熟悉，花了一些时间摸索界面
- Claude 的聊天面板有时候回复不稳定，需要刷新重试

**💡 学到的经验：**
- 不同 AI 工各有长处：ChatGPT 适合快速生成、DeepSeek 适合中文优化、Claude 适合代码相关任务
- 提示词越具体，输出质量越高
- Git 提交记录本身就是一种学习过程的可视化
- KSTAR 方法论让整个流程有章可循，不是盲目开始
